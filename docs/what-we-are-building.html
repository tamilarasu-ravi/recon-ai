What We Are Building — A Beginner's Guide

Audience: New to the project, AI cohort, or finance tech. No RAG, agents, or orchestration jargon up front — just the business problem and why the system is built the way it is.

Technical docs: For implementation detail, see README and the hero build spec.

If you were explaining this to a fresher, you would avoid talking about RAG, agents, embeddings, pgvector, evals, orchestration, etc. at the beginning.

Start with the business problem.

The Problem

Imagine you are working in the finance team of a company.

Every day employees spend money on things like:

  • AWS
  • OpenAI
  • Zoom
  • Grab
  • Hotels
  • Flights
  • Office supplies

A transaction comes in like this:

AWS
$240
Amazon Web Services

The finance team now needs to decide:

Which accounting category does this belong to?

Examples:

AWS
→ Cloud Infrastructure

Zoom
→ Software Subscription

Singapore Airlines
→ Travel Expense

Starbucks
→ Meals & Entertainment

This process is called transaction tagging or expense categorization.

Why Is This A Problem?

In many companies this is still done manually.

A finance person opens transactions one by one:

Transaction 1
Choose GL Account

Transaction 2
Choose GL Account

Transaction 3
Choose GL Account

...

Imagine doing this for:

200
500
1000
transactions

every month.

It is:

  • repetitive
  • time consuming
  • error prone

Our Goal

We want to build an AI system that helps finance teams.

Instead of:

Human
 ↓
Tag everything

we want:

AI
 ↓
Suggest category
 ↓
Human reviews only uncertain cases

This reduces manual work dramatically.

Why Not Just Ask ChatGPT?

Good question.

Many beginners think:

Transaction
 ↓
ChatGPT
 ↓
Answer

Problem:

What if ChatGPT says:

AWS
→ Travel Expense

That would be wrong.

In finance:

Wrong answer
=
Real financial mistake

So we cannot blindly trust AI.

Our Core Principle

One sentence:

A wrong automatic decision is worse than asking a human.

This is the foundation of the project.

Instead of forcing AI to answer everything:

AI always answers

we allow:

AI can say:

I know
I am not sure
I don't know

How We Solve It

We created three outcomes.

Outcome 1 — AUTO_TAG

AI is very confident.

Example:

Vendor: AWS

Past History:
50 AWS transactions

All tagged:
Cloud Infrastructure

AI says:

I am very confident.

System automatically tags it.

Outcome 2 — QUEUE_REVIEW

AI is somewhat confident.

Example:

Vendor:
OpenAI Singapore Pte Ltd

Never seen before

AI says:

Looks like Software Expense

But I am not sure.

Send to human review.

Outcome 3 — REFUSE

AI has no clue.

Example:

Vendor:
ZXQ Holdings 91827

AI says:

I don't know.

Instead of guessing:

REFUSE

and ask a human.

How Does AI Learn?

Suppose AI says:

OpenAI
→ Software Expense

Human says:

Wrong.

It should be:
AI Infrastructure Expense

The system remembers this.

Next time:

OpenAI

appears again:

AI already knows

and can classify it correctly.

This is called:

Vendor Memory

Why Use Retrieval?

Imagine seeing:

AWS Singapore

AI is unsure.

Before making a decision, we search past transactions:

AWS
AWS Marketplace
Amazon Web Services
AWS Singapore

Previously tagged as:

Cloud Infrastructure

Now AI has evidence.

Instead of relying only on its training:

Use company history

This is the RAG concept in simple form.

Why Do We Need A Policy Gate?

Companies have rules.

Example:

Any expense above $100
must have a receipt.

Employee submits:

Hotel
$300
No receipt

Even if AI knows the category:

Travel Expense

we should not auto approve it.

So:

Policy Check
 ↓
Receipt Missing
 ↓
Block Auto Tagging

This is the Policy Gate.

Why Do We Need Audit Logs?

Imagine the CFO asks:

Why did AI classify this as
Cloud Infrastructure?

We must answer.

Not:

Because AI said so.

Instead:

Vendor Rule Found

4 Similar Transactions Found

Confidence = 95%

Everything is recorded.

This is called:

Auditability

Why Do We Need Evals?

Imagine we update our prompt.

How do we know we didn't make the system worse?

We run test cases:

AWS
Zoom
OpenAI
Travel
Weird Vendors
Injection Cases

Then measure:

Accuracy
Review Rate
Refusal Rate

This is called:

Evaluation

Week 11 is heavily about this idea.

This project:

One-Line Project Summary

If an interviewer asks:

What problem does your project solve?

You can say:

Finance teams spend significant time manually categorizing transactions. This project uses AI to automatically classify expenses into accounting categories while ensuring safety through confidence scoring, human review, audit trails, and evaluation. The goal is not to replace accountants but to reduce repetitive work while maintaining financial accuracy.

That's the simplest and most accurate explanation of the entire project.
